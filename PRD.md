# 블로그치트키 - Product Requirements Document (PRD)

## 1. 프로젝트 개요

### 1.1 제품명
**블로그치트키** - AI 기반 SEO 최적화 블로그 콘텐츠 생성 플랫폼

### 1.2 제품 목적
- **문제**: 개인 사업자/마케터들이 SEO 최적화된 고품질 블로그 콘텐츠 제작에 많은 시간과 전문 지식이 필요
- **해결책**: AI 기반 자동 콘텐츠 생성으로 SEO 조건을 만족하는 설득력 있는 블로그 포스트를 빠르게 생성
- **가치 제안**: 전문 마케터 수준의 SEO 최적화 콘텐츠를 비전문가도 쉽게 제작 가능

### 1.3 타겟 사용자
- **1차**: 개인 사업자, 소상공인 (자동차 정비업, 뷰티샵, 컨설팅 등)
- **2차**: 마케팅 대행사, 콘텐츠 마케터
- **3차**: 블로거, 아프리케이트 마케터

## 2. 핵심 기능 요구사항

### 2.1 콘텐츠 생성 엔진
#### 2.1.1 키워드 분석 시스템
- **입력**: 사용자 제공 키워드 (예: "벤츠엔진경고등", "아우디A6에어컨필터")
- **처리**: Gemini 2.5 Pro를 통한 검색 의도 분석 및 부제목 제안
- **출력**: SEO 최적화된 부제목 구조 (3-5개)

#### 2.1.2 형태소 최적화 시스템
- **완전 키워드 출현**: 5-7회 정확히 유지
- **개별 형태소 출현**: 각각 15-17회 범위 내 유지
- **글자수 제한**: 공백 제외 1,500-1,700자
- **3단계 최적화**: 기본 조정 → Claude 기반 지능형 수정 → 최종 검증

#### 2.1.3 연구 데이터 수집
- **Perplexity Sonar Pro**: 신뢰할 수 있는 정보원에서 데이터 수집
- **소스 필터링**: 정부기관, 학술자료, 업계 보고서 우선
- **인용 시스템**: 자연스러운 문맥 내 연구 결과 인용

### 2.2 AI 모델 통합
#### 2.2.1 Claude Opus 4.0
- **주요 역할**: 메인 콘텐츠 생성, 편집, 최적화
- **특징**: 설득력 있는 글쓰기, 자연스러운 한국어 표현

#### 2.2.2 Gemini 2.5 Pro  
- **주요 역할**: 키워드 분석, 챗봇 대화, 콘텐츠 개선
- **특징**: 분석적 사고, 구조화된 정보 처리

#### 2.2.3 Google Vertex AI Imagen 3.0
- **주요 역할**: 인포그래픽 및 이미지 생성
- **특징**: 부제목별 맞춤 이미지, 호버 인터랙션

### 2.3 사용자 인터페이스
#### 2.3.1 프로젝트 워크플로우
```
키워드 입력 → 분석 → 연구 수집 → 비즈니스 정보 → 콘텐츠 생성 → 이미지 생성 → 편집
```

#### 2.3.2 편집 시스템
- **챗봇 인터페이스**: 자연어로 콘텐츠 수정 요청
- **실시간 미리보기**: 모바일 포맷팅 포함
- **개별 복사**: 텍스트와 이미지 별도 다운로드

### 2.4 비즈니스 정보 관리
- **재사용 가능한 프로필**: 업종, 전문성, 차별화 요소 저장
- **자동 완성**: 이전 입력 정보 기반 자동 채우기
- **다중 비즈니스**: 여러 사업체 정보 관리

## 3. 구독 및 결제 시스템

### 3.1 요금제 구조
| 플랜 | 가격 | 포함 기능 |
|------|------|-----------|
| **베이직** | 30,000원/월 | 콘텐츠 생성만 |
| **프리미엄** | 50,000원/월 | 콘텐츠 + 이미지 생성 |
| **프로** | 100,000원/월 | 풀서비스 + 챗봇 편집 |

### 3.2 결제 시스템
- **포트원(PortOne) 연동**: 정기 결제 지원
- **고객 관리**: customer_uid 기반 구독 추적
- **결제 검증**: 서버 측 결제 상태 확인

### 3.3 접근 제어
- **현재 상태**: 로그인 없이 모든 기능 무료 사용
- **향후 계획**: 로그인 시스템 활성화 후 구독 기반 제한 적용

## 4. 기술 사양

### 4.1 프론트엔드
- **프레임워크**: React 18 + TypeScript
- **라우팅**: Wouter
- **상태관리**: TanStack Query
- **UI 컴포넌트**: Radix UI + Tailwind CSS
- **빌드 도구**: Vite

### 4.2 백엔드
- **런타임**: Node.js + Express
- **언어**: TypeScript (ES 모듈)
- **데이터베이스**: PostgreSQL + Drizzle ORM
- **세션**: PostgreSQL 기반 세션 스토어

### 4.3 AI 서비스 연동
- **Anthropic API**: Claude 4.0 Sonnet/Opus
- **Google AI**: Gemini 2.5 Pro, Vertex AI Imagen 3.0
- **Perplexity API**: Sonar Pro 모델

### 4.4 배포 환경
- **개발**: Vite Dev Server + tsx
- **프로덕션**: Replit Deployments
- **도메인**: `.replit.app` (커스텀 도메인 연결 가능)

## 5. 품질 기준

### 5.1 SEO 최적화 기준
- **키워드 밀도**: 완전 키워드 5-7회, 개별 형태소 15-17회
- **글자수**: 1,500-1,700자 (공백 제외)
- **구조**: 서론(35-40%) + 본론 + 결론
- **가독성**: 모바일 친화적 줄바꿈 (25-30자)

### 5.2 콘텐츠 품질 기준
- **설득력**: 독자 공감 기반 서론, 자연스러운 CTA 결론
- **신뢰성**: 연구 데이터 기반 주장, 출처 명시
- **자연스러움**: 대화체 어투, 비즈니스 전문성 자연스럽게 통합

### 5.3 성능 기준
- **생성 속도**: 평균 2-3분 내 완성
- **정확도**: SEO 조건 90% 이상 1차 생성 성공률
- **사용자 경험**: 단계별 진행 상황 실시간 표시
